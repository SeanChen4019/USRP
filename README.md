# USRP 无线图像/视频传输演示平台

基于 MATLAB 和 Ettus X310 USRP 的**分块渐进式无线传输演示系统**。将图像和视频预切分为独立块，通过 USRP 射频链路发送块索引与校验码，接收端逐块恢复显示，缺失块黑色填充，直观展示无线信道特性。

## 系统架构

```
发射端 (发射机/)                     接收端 (接收机/)
┌─────────────────────┐             ┌─────────────────────┐
│ 预处理图片/视频为块   │             │ 预处理图片/视频为块   │
│ (8×8 图像块 + 视频帧) │             │ (预存全部块数据)      │
├─────────────────────┤             ├─────────────────────┤
│ 块元数据 → LDPC编码  │   USRP     │ LDPC译码 → 块元数据   │
│ QPSK/BPSK 调制      │  ╔════╗    │ QPSK/BPSK 解调       │
│ 升余弦滤波           │──║空口║──→│ 升余弦滤波            │
│                     │  ╚════╝    │                      │
├─────────────────────┤             ├─────────────────────┤
│ 固定2轮发送 + 空闲帧  │             │ CRC校验 → 逐块点亮    │
│ 发完自动停止          │             │ 缺失块 = 黑色填充     │
└─────────────────────┘             └─────────────────────┘
```

## 快速开始

### 1. 环境准备

确保两台 PC 分别连接一台 X310 USRP，配置外部时钟源，IP 地址设为 `192.168.10.2`。

### 2. 发射端

在 MATLAB 中运行：
```matlab
cd 发射机
Transmitter_Main_UDPui0416
```

发射机将自动：
- 加载 `p2.jpg` 和 `视频.mp4`
- 图片切分为 8×8 = 64 块
- 视频提取 20 个均匀关键帧
- 通过 USRP 发送 2 轮块数据
- 发完后自动停止

### 3. 接收端

在 MATLAB 中运行：
```matlab
cd 接收机
Receiver_main_UDPui0416
```

接收机将自动：
- 预加载相同的图片和视频块数据
- 接收 USRP 信号，逐块 CRC 校验
- 左侧显示图片恢复进度，右侧显示视频帧恢复进度
- 缺失块以黑色填充

## 关键参数

在 `Transmitter_Main_UDPui0416.m` 和 `Receiver_main_UDPui0416.m` 顶部可调整：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `NUM_ROUNDS` | 2 | 固定发送轮数（非无限循环） |
| `IDLE_SLOTS` | 3 | 每发一组后的空闲帧数（给接收机处理时间） |
| `IMAGE_GRID_ROWS` | 8 | 图片分块行数 |
| `IMAGE_GRID_COLS` | 8 | 图片分块列数 |
| `VIDEO_FRAME_NUM` | 20 | 视频提取帧数 |
| `BURST_PKTS` | 10 | 每突发包含的包数 |
| `Threshold` | 240 | 同步检测门限 |

## 物理层参数

| 参数 | 值 |
|------|-----|
| 调制方式 | QPSK (常规) / BPSK+扩频SF15 (抗扰模式) |
| 信道编码 | LDPC (648, 486)，码率 ~0.75 |
| CRC | CRC-32 (Ethernet 标准多项式) |
| 成形滤波 | 升余弦，滚降系数 0.25，4倍过采样 |
| 帧同步 | 511位 PN 序列滑动相关 |

## 数据包格式

每个物理包通过 LDPC + CRC 保护，载荷 44 字节：

| 字节偏移 | 字段 | 位宽 | 说明 |
|---------|------|------|------|
| 0 | block_row | 8 | 块行索引 (0-based) |
| 1 | block_col | 8 | 块列索引 (0-based) |
| 2 | total_rows | 8 | 总行数 |
| 3 | total_cols | 8 | 总列数 |
| 4 | block_type | 8 | 0=图像块, 1=视频帧 |
| 5-8 | crc32 | 32 | 块数据 CRC32 校验 |
| 9-43 | 保留 | — | 零填充 |

## 演示效果

- **图片**：8×8 网格从全黑开始，逐块"点亮"，2 轮后丢失率 < 10%（丢包率 30% 时）
- **视频**：20 帧缩略图横向排列，逐帧恢复
- **缺失块**：黑色填充，直观展示无线信道丢包特性
- **运行时间**：约 10-30 秒完成传输（取决于 IDLE_SLOTS 设置）

## 设计理念

1. **非无限广播**：固定轮次发送，发完自动停止
2. **无需反馈链路**：纯前向传输，不依赖反向链路（USRP 同子板无法同时收发）
3. **传输压力小**：每包仅传块索引+CRC（8字节有效载荷），而非完整图像数据
4. **优雅降级**：缺失块黑色填充，不影响整体画面辨识
5. **收发速率匹配**：空闲帧机制给接收机 LDPC 译码处理时间
